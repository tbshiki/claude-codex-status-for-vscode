# 開発ワークフロー

## 1. 現状確認

`git status --short`、関連ファイル、リポジトリ内の指示を確認し、既存変更と生成物を識別する。

## 2. タスク分解

依存関係と検証方法を踏まえて、小さく確認可能な単位へ分ける。対象外を明確にする。

## 3. 実装

既存規約に合わせ、正典となるファイルだけを変更する。無関係な整形やリファクタリングを混ぜない。

## 4. 段階的な検証

構文確認、対象テスト、静的解析、より広いテスト、ビルドの順に、利用可能なものを軽量なものから実行する。

## 5. 最終差分確認

`git diff`、`git status --short`、`git diff --check` を確認し、意図しない変更や秘密情報がないことを確かめる。

## 6. 未確認事項の報告

変更内容、成功した検証、実行できなかった検証と理由、残るリスク、必要な手動作業を簡潔に報告する。
